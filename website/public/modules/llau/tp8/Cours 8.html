Que trouve-t'on dans Keras ?

Keras est une API de deep learning, interfacable avec plusieurs framework comme Tensorflow, CNTK, Theano ou MxNet

Points positifs :

  • Facilit de prise en main, user-friendly, ressemble sklearn
  • Modulaire : on construit le graphe couche par couche ou comme un graphe
  • Permet de mettre en place rapidement des tests
  • Grosse communaut donc plein de papiers implments
  • Plus rapide que torch en prdiction

On trouve quatre activations principales :

  • linear : le signal n'est pas transform
  • tanh : $\frac{e^x1}{e^x+1}$retourne une valeur entre -1 et 1
  • sigmoid : $ f(x)=\frac{1}{1 + e^{- x}}$, retourne une valeur entre 0 et 1
  • relu : $max(0,x)$, on garde uniquement ce qui nous intresse

La denire couche du rseau est celle qui nous donne l'output dsir. Sa dimension, ainsi que son activation doivent tre choisi pour rpondre la problmatique voulu.

Classification binaire :

  • nombre de neurones : 1
  • activation : sigmoid $ f(x)=\frac{1}{1 + e^{- x}}$
  • loss : binary_crossentropy

Classification multiclasses :

  • nombre de neurones : nombre de classes
  • activation : softmax
  • loss : categorical_crossentropy

Rgrssion :

  • nombre de neurones : 1
  • activation : linaire
  • loss : toute loss diffrentiable

Le choix de l'algorithme d'optimisation est toujours une grande question. L'utilisation de l'algorithme peut tre ralis de deux faons.

In[18]:
from keras.optimizers import SGD
#Avec les paramtres par dfault

## Ou bien pour pouvoir customiser l'algorithme
sgd = SGD(lr=0.01, decay=1e-6, momentum=0.9, nesterov=True)
model.compile(loss='mean_squared_error', optimizer=sgd)

Voici un rapide rcapitulatif des principaux algorithmes d'optimisation :

  • SGD, le plus classique, fluctue autour du minimum
  • RMSprop, bon choix pour les rseaux rcurrent
  • Adam, efficace dans la majorit des cas (https://arxiv.org/abs/1412.6980v8)

La couche d'input change pour tre un embedding, c'est--dire une couche de vectorization permettant d'utiliser des donnes textuelles.

Word embedding: technique permettant de reprsenter chaque mot d'un dictionnaire par un vecteur de nombres rels.

2 phrases:

  • "Hope to see you soon"
  • "Nice to see you again" On assigne un entier unique chaque mot, dans l'ordre d'apparition.
  • [0, 1, 2, 3, 4]
  • [5, 1, 2, 3, 6]

Si on veut entrainer notre reseau sur cette premire couche il faut alors spcifier: Embedding(7, 2, input_length=5) 7 correspond au nombre de token distincts (vocab size), 2 indique la dimension de l'embedding, et l'input_length la longueur de chaque sequence.

def VGG16_model(input_shape=(224, 224, 3)):
    model = Sequential()

    # Block 1
    model.add(Conv2D(64, (3, 3), activation='relu', padding='same', input_shape=input_shape))
    model.add(Conv2D(64, (3, 3), activation='relu', padding='same'))
