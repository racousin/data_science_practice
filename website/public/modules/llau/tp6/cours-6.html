1.2 Boosting & XtremBoosting

Les trois algorithmes dcrits ci-dessous sont des mthodes de gradient boosting. LightGBM et Xgboost sont des implmentations dites d'extrme boosting, c'est--dire qu'elles sont construites de manire utiliser du mieu possible les ressources computationnelles et d'un point de vu thorique la diffrentielle est d'ordre 2.

Il est important de not que ces mthodes sont implments de manire propre mais possde galement un wrapper scikit-learn.

Un autre atout majeur de ces implmentations est l'utilisation de jeux de validation pour obtenir le nombre d'itrations optimal. A chaque itration on regarde si les performances sur le jeu de validation sont amliors, sinon on arrte. On prend alors la dernire itration la plus performante. Evidemment, il est possible de choisir un nombre d'itration pour lequel il n'y a d'amliorations. (voir Tp)

1.2.1 XGBoost

Original paper (T.Chen, 2016) : https://arxiv.org/abs/1603.02754

Il existe plusieurs diffrences notable avec l'implmentation du GBT classique :

  • Ne tiens pas compte des valeurs manquantes...
  • Le modle est mieux rgularis, on contrle mieux l'overfitting, ce qui explique ces bonnes performances
  • On trouve des paramtres de pnalisation $L^1$ et $L^2$, qui n'existe pas dans la version original
  • D'autres methodes que les arbres CART peuvent tre utiliss: des rgrssions linaire (Linear Boosting), ou des arbres DART (Dropouts meet Multiple Additive Regression Trees, http://proceedings.mlr.press/v38/korlakaivinayak15.pdf)
  • On peut rsoudre la grande majorit des problmatique industrielles avec cette implmentation, de la rgrssion au ranking, en passant par de la classification multi-classe.
  • Trs customisable

On retrouve les mme paramtres que pour l'implmentation du GBT dans sklearn. Attention, le nom n'est pas forcment le mme, je vous invite lire la doc. Cependant, il en existe quelques autres qui sont importants:

  • booster : type de mthode booster
  • objective : l'objectif du modle (rgssion, etc.)
  • tree_method : mthodes de construction des arbres (exact, approx, histogramme)
  • eval_metric : choix de la mtrique d'valuation

Pour utiliser la version sklearn : "from xgboost import XGBRegressor" (ou XGBClassifier).

/!\ Tous les paramtres ne sont pas disponibles dans le wrapper sklearn.

1.2.3 CatBoost

Original paper (Yandex, 2017) : https://arxiv.org/abs/1706.09516

Dernier n de la firme russe Yandex, CatBoost est une implmentation similaire celle de xgboost et de lightgbm mais qui a la particularit de tenir compte des variables catgorielles pour l'apprentissage. L'algorithme va construire partir des variables catgorielles diverses statistiques* et, tenir compte de celles-ci pour l'apprentissage. On trouve quelque diffrences notables avec les deux autres algorithmes prsents ci-dessous:

  • Trs performant mais trs lent.
  • Pas d'autre booster que les arbres de disponibles
  • Deux taches : rgrssion ou classification
  • Plusieurs paramtres pour l'apprentissage des variables catgorielles.
  • Paramtres de dtction de l'overfitting
  • Il y a une interface graphique super sexy...

*Voir https://tech.yandex.com/catboost/doc/dg/concepts/algorithm-main-stages_cat-to-numberic-docpage/

Quelques paramtres importants :

  • iterations : nombre d'arbres
  • eval_metric : la mtrique d'valuation
  • ctr : paramtre de transformation des variables catgorielles
  • fold_permutation_block_size : nombre de permutations des catgorielles

Pour utiliser la version sklearn : "from catboost import CatBoostRegressor" (ou CatBoostClassifier). Et pour utiliser l'apprentissage sur les variables catgorielles, ajouter dans fit cat_features = [index des features catgorielles].

/!\ Tous les paramtres ne sont pas disponibles dans le wrapper sklearn.

2.1.0 Dfinition

On parle de feature importance presque uniquement dans le cas des algorithmes de machine learning bas sur les arbres CART. L'importance d'une variable, n'est autre que le poids - selon une mesure donne - qu'apporte cette variable dans l'apprentissage. Pour les arbres CART on retrouve la dfinition suivante :

*Les arbres alatoires sont une rgle de dcision interprtable pour la prdiction des classes.En effet, les variables de dcoupe sont choisies selon leur capacit classifier les donnes. De ce fait, une variable permettant de dcouper les premiers nuds a un pouvoir discriminant plus important quune variable apparaissant dans les dernires dcoupes (ou napparaissant pas dans larbre). * B.Gregorutti- https://tel.archives-ouvertes.fr/tel-01146830/document

Dans les mthodes d'ensemble, l'importance des features est calcul gnralement de la manire suivante: On compte le nombre de fois o la variable est slctionn pour sparer un noeud, pondr par l'amlioration du modle chaque sparation. Puis on moyenne les rsultats.

Pour rsumer : variable trs utilise qui spare bien les donnes = variable importante.

2.1.1 Permutation importance

La permutation importance est une autre mthode pour calculer l'importance des features, elle a les particularits suivantes:

  • Calcul rapide
  • Beaucoup utilise et comprise
  • Consistante

Comment a marche ?

    1. Dcouper en train test
    1. Faire apprendre un modle sur nos donnes
    1. Mesurer les performances du modle
    1. Mlanger les valeurs d'une colonne d'un jeu de test puis prdire
    1. Evaluer de nouveau les performances

Plus la mtrique souffre de ce changement plus la variable est importante.

-> Remettre en place la colonne, puis rpter sur toutes les variables du jeu de donnes

2.1.2 LOFO

LOFO (Leave One Feature Out) Importance calcule l'importance pour un ensemble de variable, bas sur une mtrique choisi, avec un modle choisi. Itrativement une variable va tre enlev de l'ensemble puis on va rvaluer les performances du modles avec un schma de validation choisi.

https://github.com/aerdem4/lofo-importance

Avantages :

  • Se gnralise bien aux donnes non observes
  • Indpendant du modle
  • Donne une importance ngative aux variables qui dgrade les performances en les incluants.

2.2 - Feature Selection

Il est parfois ncssaire d'effectuer une slction des variables utilises dans le modle pour plusieurs raisons:

  • Trop de corrlations entre certaines variables

  • Expertise mtier

  • Et enfin, importance faible

Il existe alors plusieurs technique pour slctionner les variables, autre que les critres d'Akkaike etc.

        for regressor in regressor_list:
            model = regressor['reg']
            t0 = time.time()
            try:
                model.fit(x_train, y_train)
            except ValueError as err:
                print(f'{err}')
            t1 = time.time()
            preds = model.predict(x_test)
            print(f"{regressor['name']} apprend en {t1-t0:.3f}s et a une performance de {rmse(y_test, preds):.2f}")
